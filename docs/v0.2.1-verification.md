# v0.2.1 Verification Record

## Release Boundary

v0.2.1 adds the Localization Mode System with four operating modes,
reference policy enforcement, existing-locale preservation behavior,
and obsolete target-only key protection.

## Modes Verified

| Mode | Reference Policy | Status |
|------|-----------------|--------|
| `blind_benchmark` | `blind` | pass |
| `greenfield_localization` | `style_only` | pass |
| `existing_locale_maintenance` | `preserve_existing` | pass |
| `rewrite_or_harmonization` | `tm_assisted` | pass |

## Key Pass Criteria

### Blind Benchmark Isolation

- Generation-facing artifacts (work packets, draft requests, prompts,
  generated batches) must not leak existing target translations into
  blind generation workflows.
- Negative check: `blind_artifact_with_existing_target_translation_is_rejected` — pass.

### Existing Locale Maintenance

- Unchanged reviewed target translations are preserved, not mass-rewritten.
- Only new/missing source segments generate translation work.
- Stale target segments (source changed since last review) are marked
  stale or review-required.
- Maintenance mode without `rewrite_or_harmonization` triggers a mass-rewrite
  rejection negative test — pass.

### Obsolete Target-Only Preservation

Sentinel: `legacy_removed_key` = `旧版专属译文_不得自动删除`

| Check | blind | greenfield | maintenance | rewrite |
|-------|-------|-----------|-------------|---------|
| `detected_as_obsolete` | true | false | true | true |
| `present_in_original_target` | true | false | true | true |
| `present_in_staged_target` | false* | false | true | true |
| `present_after_apply_to_copy` | false* | false | true | true |
| `deleted_by_apply_plan` | false | false | false | false |
| `generation_facing_leakage` | false | false | false | false |
| `requires_owner_review` | true | false | true | true |

*blind_benchmark correctly excludes obsolete keys from generation;
they are absent by policy, not by deletion.

### Source Mutation Check

Source fixture file hashes unchanged before and after all four mode runs.

### Same-Project Behavior Difference

Same fixture produces all-candidate blind plan (12 generated) and
preserve-first maintenance plan (2 generated, 10 preserved).

### Negative Checks

| Check | Result |
|-------|--------|
| Blind artifact with existing target translation rejected | pass |
| Maintenance mass rewrite without rewrite mode rejected | pass |
| Maintenance staged file missing obsolete key rejected | pass |
